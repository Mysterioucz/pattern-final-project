<objective>
Fix data leakage in `load_snapshots()` (CR-03) and update the deprecated `DataLoader`
import in `loader.py` (WR-04). Also correct the off-by-one feature count in the
`load_snapshots()` docstring (IN-03: says 166 features, actual is 165).

Purpose: `load_snapshots()` currently fits a fresh `StandardScaler` inside the
per-snapshot loop for every timestep, including test snapshots (t >= 35). This means
test-set feature statistics are used to normalize test data — violating temporal
evaluation integrity. `load_static_graph()` already implements the correct pattern
(fit on train rows, transform all rows); `load_snapshots()` must mirror it.

Output: A `load_snapshots()` that fits the scaler once on all training-split rows
(t 1-34 pooled) before the loop, then calls `scaler.transform()` for each snapshot.
Loader.py updated to use the current PyG import path.
</objective>

Insert this block:

    # Fit scaler once on all training-split rows (t < TEST_START_TIMESTEP).
    # Mirrors load_static_graph() which fits only on train nodes to prevent
    # test-set statistics leaking into normalisation (CR-03 fix).
    if normalize:

**Change 3 — Replace fit_transform with transform inside the per-snapshot loop (CR-03):**
Inside the loop, replace the current conditional block:

    BEFORE (lines ~355–357):
        if normalize:
            scaler = StandardScaler()
            x_np = scaler.fit_transform(x_np)

    AFTER:
        if normalize:
            x_np = scaler.transform(x_np)

This removes the per-snapshot scaler instantiation and fit. The `scaler` variable is
now the one fitted on all training rows before the loop; it is reused for every
snapshot including test ones.

**Why this mirrors load_static_graph():**
load_static_graph() (lines ~274–277) does:
    scaler = StandardScaler()
    x_np[train_mask_np] = scaler.fit_transform(x_np[train_mask_np])
    x_np[test_mask_np] = scaler.transform(x_np[test_mask_np])

load_snapshots() pools all training timestep rows to fit the scaler, then applies
scaler.transform() per-snapshot — same principle, adapted for the per-snapshot loop
structure.

**Important edge case:** The `scaler` variable is only defined when `normalize=True`
(the guard `if normalize:` wraps the fit). The per-snapshot `if normalize: x_np =
scaler.transform(x_np)` branch is only reached when normalize=True, so the variable
is always in scope when used. No change needed for the `normalize=False` path.
